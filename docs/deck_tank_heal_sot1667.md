# 重量級タンク回復デッキの構築と champion_v2 判定 (SOT-1667, D2)

SOT-1665（ルールベース 竹 対 梅）承認済み D 系列計画の D2。梅 agent（R3 RuleAgent）の
長期戦・リソース管理力を活かす「重量級タンク＋回復コントロール」デッキを構築し、
デッキ最適化トラック基盤（SOT-1651: `eval/deck_eval.py` の paired A/B）で現行 champion
（`decks/champion_v1.csv`）比の性能を実測して champion_v2 昇格の採否を判定した。

## 結論 — champion_v2 昇格は見送り、現行 champion v1 を維持

タンク回復型は 2 案（v1: 回復高密度／v2: 改良再試行・高インパクト回復のみ）とも、
現行 champion に対し **プール全体で有意に弱い**（Wilson 95% CI の上限が 0.5 未満）。
受け入れ条件の「有意に弱い場合は 1 回まで改良再試行し、それでも劣るなら現行維持を
正直報告」に従い、現行維持と判定する。`decks/registry.json` は変更しない
（`current: v1` のまま。昇格しない挑戦者を champion 版管理には登録しない）。

## なぜタンク回復型が現行に勝てないか（根拠）

1. **主砲がエネルギー密度そのもの。** Mega Abomasnow の Hammer-lanche は「山札上 6 枚を
   トラッシュし、その中の基本水エネ 1 枚につき 100 ダメージ」— 期待値はデッキのエネ比率
   ×600。エネ 35→28（v1）で期待 350→280、35→33（v2）でも 350→330 と、回復スロットは
   そのまま打点低下になる。350HP の Mega ex 同士のミラーでは期待 350 が KO ラインを跨ぐ。
2. **固定 agent はサポーターを等価に扱う。** RuleAgent の MAIN 方策は PLAY 候補を同点
   （400 点）でランダム選択するため、回復サポーターは無傷の盤面でも 1 ターン 1 回の
   サポーター枠を Lillie's（ドロー）/ Waitress（エネ加速）から奪う。回復の期待効用が
   打点・テンポの損失を下回る。
3. SOT-1662（竹側の候補デッキ比較）の「エネルギーを薄くした候補は有意に弱い」教訓と
   整合する再確認となった。
